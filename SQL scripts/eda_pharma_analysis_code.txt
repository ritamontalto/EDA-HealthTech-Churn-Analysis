
- I start my analysis by looking at retention rates, thinking of the following questions/business problems


--- Q1: How many users left the program? 

SELECT 
	clean_user_churn,
	COUNT(*) AS users
FROM clean_users
WHERE clean_user_churn <> 'Unknown'
GROUP BY clean_user_churn
ORDER BY users DESC;



--- Q2: How many users are in each engagement category and which activity groups have the highest churn rate? 

SELECT 
	clean_user_activity,
	clean_user_churn,
	COUNT(*) AS users
FROM clean_users
WHERE clean_user_activity <> 'Unknown' AND clean_user_churn <> 'Unknown'
GROUP BY clean_user_activity, clean_user_churn
ORDER BY clean_user_activity, clean_user_churn DESC;



--- Q3: Which age segments are leaving the program the most?

SELECT
    c.age_range,
    u.clean_user_churn,
    COUNT(*) AS users
FROM clean_users AS u
JOIN clean_customers AS c
    ON u.user_id = c.user_id
WHERE clean_user_churn <> 'Unknown'
GROUP BY c.age_range, u.clean_user_churn
ORDER BY c.age_range, u.clean_user_churn DESC;



--- Q4: Which diabetes population is hardest to retain?

SELECT
	clean_diagnosis,
	clean_user_churn,
	COUNT(*) AS users
FROM clean_users
WHERE clean_diagnosis <> 'Unknown' AND clean_user_churn <> 'Unknown'
GROUP BY clean_diagnosis, clean_user_churn
ORDER BY clean_diagnosis, clean_user_churn DESC;




- then I decided to focus on understanding the reasons why users leave, through the following questions


--- Q5: Are less engaged users more likely to leave the program? 

SELECT 
clean_user_activity,
COUNT(*) AS total_users,
SUM(CASE
        WHEN clean_user_churn = 'Yes' THEN 1
        ELSE 0
        END
    ) AS churned_users,
ROUND(SUM(CASE
              WHEN clean_user_churn = 'Yes' THEN 1
              ELSE 0
              END
          ) * 100.0 / COUNT(*), 2
     ) AS churn_rate_percentage
FROM clean_users
WHERE clean_user_churn <> 'Unknown' AND clean_user_activity <> 'Unknown'
GROUP BY clean_user_activity
ORDER BY churn_rate_percentage DESC;



--- Q6: Are retained users showing stronger long-term diabetes control?

SELECT
    clean_user_churn,
    ROUND(CAST(AVG(initial_hba1c) AS NUMERIC), 2) AS avg_initial_hba1c,
    ROUND(CAST(AVG(current_hba1c) AS NUMERIC), 2) AS avg_current_hba1c,
    ROUND(CAST(AVG(initial_hba1c - current_hba1c) AS NUMERIC), 2) AS avg_hba1c_reduction
FROM clean_users
WHERE clean_user_churn <> 'Unknown'
GROUP BY clean_user_churn
ORDER BY clean_user_churn DESC;
--- AVG was returning a double precision value, so I cast it to numeric before rounding to two decimal places.



--- Q7: Are users who improve their glucose control more likely to remain enrolled?

SELECT
	clean_user_churn,
	ROUND(CAST(AVG(initial_glycemia_rate) AS NUMERIC), 1) AS avg_initial_glycemia,
	ROUND(CAST(AVG(final_glycemia_rate) AS NUMERIC), 1) AS avg_final_glycemia
FROM clean_users
WHERE clean_user_churn <> 'Unknown'
GROUP BY clean_user_churn
ORDER BY clean_user_churn DESC;





- I also decided to analyze treatment effectiveness for the users, through these questions


--- Q8: Which patient groups benefit most from the digital therapeutic program and experience the greatest improvement in long-term diabetes control?

SELECT
	clean_diagnosis,
	ROUND(CAST(AVG(initial_hba1c - current_hba1c) AS numeric), 2) AS avg_hba1c_reduction
FROM clean_users
WHERE clean_diagnosis <> 'Unknown'
GROUP BY clean_diagnosis
ORDER BY avg_hba1c_reduction DESC;



--- Q9: Does higher engagement translate into better clinical outcomes?

SELECT
	clean_user_activity,
	ROUND(CAST(AVG(initial_hba1c) AS numeric), 2) AS avg_initial_hba1c,
	ROUND(CAST(AVG(current_hba1c) AS numeric), 2) AS avg_current_hba1c
FROM clean_users
WHERE clean_user_activity <> 'Unknown'
GROUP BY clean_user_activity
ORDER BY avg_initial_hba1c DESC;





- Finally, I decided to get some useful business insights, so I thought of the following business problems


--- Q10: Which enrollment channels bring the most users into the program?

SELECT
	clean_enrollment_channel,
	COUNT(*) AS users
FROM clean_users
WHERE clean_enrollment_channel <> 'Unknown'
GROUP BY clean_enrollment_channel
ORDER BY users DESC;



--- Q11: Are some acquisition channels attracting users with higher churn risk?

SELECT
	clean_enrollment_channel,
	clean_user_churn,
	COUNT(*) AS users,
	ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(PARTITION BY clean_enrollment_channel), 2) AS percentage
FROM clean_users
WHERE clean_enrollment_channel <> 'Unknown' AND clean_user_churn <> 'Unknown'
GROUP BY clean_enrollment_channel, clean_user_churn
ORDER BY clean_enrollment_channel, clean_user_churn DESC;



--- Q12: Which treatment plans achieve better retention?

SELECT
    clean_treatment_plan,
    clean_user_churn,
    COUNT(*) AS users
FROM clean_users
WHERE clean_treatment_plan <> 'Unknown' AND clean_user_churn <> 'Unknown'
GROUP BY clean_treatment_plan, clean_user_churn
ORDER BY clean_treatment_plan, clean_user_churn DESC;



--- Q13: How long do users stay in the program on average before leaving?

SELECT
	clean_user_churn,
	ROUND(CAST(AVG(months_in_program) AS NUMERIC), 1) AS avg_months
FROM clean_users
WHERE clean_user_churn <> 'Unknown'
GROUP BY clean_user_churn
ORDER BY clean_user_churn DESC;



- I decided to remove all null or unknown values from every query because if not, I might include categories that aren't useful or don't represent the actual status, especially in some specific queries. Needless to say, null values can also be an important insight, as a matter of fact I will include them on data visualization.
